StoryMap.io

An agile tool using Github to map project management. A user story map arranges user stories into a useful model to help understand the functionality of the system, identify holes and omissions in your backlog, and effectively plan holistic releases that deliver value to users and business with each release - Jeff Patton in The New User Story Backlog Is a Map.

StoryMap was developed at Univeristy of Alberta with Aaron Yong, Eleni Stroulia, Elvis Lin, Seth Davis, and myself for CMPUT 402-Software Quality. The goal of the project was to provide a story mapping tool for GitHub projects. In sense an abstraction layer for Github issues that allows them to fit within scrum planning.
